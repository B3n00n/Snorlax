System Overview

Snorlax is a three-part system designed to make Quest management simple and efficient.

Snorlax Setup

Purpose: Automated setup tool for Quests

  • One-click installation process
  • Handles ADB configuration
  • Sets device owner permissions
  • GUI application

Snorlax Client

Purpose: Background service running on Quest

  • Receives remote commands
  • Manages app installations
  • Reports battery status
  • Executes system operations

Snorlax Server

Purpose: PC GUI control interface

  • Manages multiple Quest devices
  • Real-time device monitoring
  • Batch operations support
  • Cross-platform compatibility

Initial Setup Requirements

Prerequisites

Component Requirement Notes
Quest Device Factory Reset State Must be freshly reset for device owner setup
Developer Mode Enabled Required for ADB access
Network Internet connection Quest needs an internet connection since it downloads Snorlax APK from the cloud. PC doesn't matter
ADB Tools Platform Tools Auto-detected or manually specified

Network Configuration

  • Default server port: 8888 (TCP)
  • HTTP server port: 8889 (for local APK hosting)
  • Make sure these ports are unused/not blocked by firewall

Part 1: SnorlaxSetup Tool

The Windows application that automates the entire Quest setup process.

Setup Process

  1. Connect Quest via USB
    Connect a Quest via USB
  2. Launch SnorlaxSetup.exe
    The tool will auto-detect the connected device (Only quest devices will get detected, emulators may shit on this detection, so close them if opened prior)
  3. Verify Device Status
    Check that all indicators show green/ready state
  4. Click "Start Setup"
    The tool will download and install Snorlax automatically
  5. Wait for Completion
    Process takes 30 seconds ish depending on connection, Snorlax size and apk installation on device.

Advanced Settings

ADB Path Configuration: - Default: C:\platform-tools\adb.exe - Custom path can be set via Advanced button - Tool auto-detects common ADB locations

Part 2: Snorlax App on Quest

The background service that enables remote control of Quest devices.

How It Works

Auto-Start Service

Snorlax starts automatically on boot and runs in the background. No user interaction needed!

Device Owner Mode

Special Android permissions allow silent app installation and system management.

Network Connection

Maintains persistent connection to control server.

Configuration

Server IP can be changed via the config screen (appears when launching through device notifications).

Changing Server Configuration

  1. Launch Snorlax app from Quest notifications (if needed to change server)
  2. Enter new server IP address
  3. Enter server port (default: 8888)
  4. Click "Save & Apply"
  5. Service will restart with new settings

Starting the Server

  1. Launch SnorlaxServer.exe
  2. Server → Start Server (from menu) - Should start automatically, but anyway
  3. Devices will connect automatically considering they are configured properly (Same LAN, proper IP configured)
  4. Green status = Server Running
